In a [previous post](#/blog?id=safety-align-as-world-reasoner), I argued that truly robust safety alignment requires models to become world reasoners: models capable of predicting how their actions change states of the world, rather than recognizing prohibited linguistic forms. Models trained only to avoid forbidden scripts learn discriminative boundaries over language; they do not internalize the causal structure of why certain outcomes should not be brought about. This is why surface-level alignment appears stable within familiar distributions yet degrades rapidly under modest distribution shifts: pattern matching is not consequence reasoning. That post ended by gesturing toward constructing minimal world-interacting agents as a practical intermediate path, somewhere between the pure text world, which cannot support genuine consequence reasoning, and fully embodied interaction, which remains distant. This post is about one concrete instance of that path.

The fundamental problem with supervising safety in a pure text environment is that the supervision signal is disconnected from the thing we actually care about. We care about whether a model's output makes harmful events more achievable. But in a text-only setting, consequences are described, not produced. LLM-as-judge systems evaluate linguistic plausibility. Human labelers are inconsistent on borderline cases. Refusal templates can be paraphrased around. None of these signals directly measure whether a model has correctly reasoned about world states; they measure whether its output resembles what safety-trained text looks like. The result is the same failure mode I described before: models learn to perform consequence reasoning rather than engage in it. The pure text world is a low-cost, low-constraint environment where safety alignment readily collapses into a linguistic game.

Code has a property that natural language fundamentally lacks: executability. When a model writes code, and that code is run, something happens. The environment responds. A function returns a value. A simulation advances a state. The output is not a description of a consequence; it is a consequence, minimal and contained, but real within its environment. This is categorically different feedback from asking another language model whether an output seemed dangerous.

Code is also verifiable and interactive in ways that prose is not. Whether a program achieves its stated goal can be determined by running it, not by soliciting a judge's opinion. A model working in a code environment must act, observe the result, and respond, participating in a feedback loop where actions have deterministic effects. This structure is much closer to the kind of world interaction that consequence reasoning requires than any single-turn question-and-answer format. Code is, in short, a minimal world: controllable, feedback-rich, and verifiable, exactly the kind of environment that can ground supervision in something outside the model's own output distribution.

The natural objection is that most harmful requests are not naturally expressed as coding tasks. "How do I synthesize this compound?" is not a programming problem. Translating every harmful goal mechanically into code would be artificial and often impossible. This is correct. But I think it misidentifies where the insight actually lives.

Harmful events are almost never single-step processes. They unfold through chains of enabling actions: gathering and processing information, transforming it into usable form, routing it through one system toward another, connecting one capability to the next. Many of these intermediate steps, even when the overall goal is non-technical, involve structured, verifiable computational tasks: identifying a relevant entity, filtering a dataset, organizing outputs into a usable format, constructing a query. The overall causal chain may not be a programming problem, but a causally critical link within it often can be.

If a model is placed at a causally relevant intermediate step of a harmful process, and that step can be expressed as a verifiable code task, then whether the model completes that task is a meaningful signal. It is not a signal about whether the model produced text that sounds relevant to harm. It is a signal about whether the model took an action that materially advances a harmful event. Completing the step does not produce dangerous-sounding text; it produces a working artifact that moves the causal chain forward. That is a different kind of evidence.

This is what I mean by event-level supervision. The unit of analysis is not the utterance but the event: the causal progression from a harmful intent through a series of enabling steps toward an outcome. A model that successfully navigates an executable intermediate step of a harmful process is doing something categorically different from a model that merely writes about that process. The difference is causal contribution, not linguistic resemblance. To supervise the former, you need a substrate that can register causal contribution. Text cannot do this. Code, within its limits, can.

Rather than asking whether a model's output resembles safe text to a language-based evaluator, we ask whether a model's action advances a harmful event in a verifiable way. Code environments are a substrate where that question can be posed and, in meaningful cases, answered. They are not the world, but they are closer to the world than language alone.

The broader argument remains the same as before: alignment that is robust requires models to reason about how their actions change world states, not just recognize patterns over linguistic forms. Code environments are one concrete step toward making that standard supervisable, not because code captures the full complexity of world states, but because it introduces the minimal structure world interaction requires: action, feedback, and observable consequence. Code is the bridge. Building more of these bridges, and understanding where they hold and where they break, is where I think the next progress in consequence-grounded safety supervision will come from.
